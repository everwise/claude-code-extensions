# Agent Usage Guide

> For the full agent list and overview, see the [main README](../README.md).

## Invocation

```bash
> use [agent-name] for [specific-task]
> use [agent-name] to [action] [target]
```

### Examples

```bash
> use jira-workitem-implementer for APL-1234
# → Analyzes → Plans → Implements → Tests → Creates PR

> use feature-architect to implement OAuth authentication
# → Explores codebase → Plans approach → Guides implementation

> use debugger to investigate failing tests in UserService
# → Analyzes → Identifies root cause → Provides fix

> use iterative-quality-fixer to resolve linting errors in src/auth/
# → Fix → Test → Verify → Repeat until passing

> use pr-review-comment-resolver for PR #123
# → Fetches comments → Categorizes → Systematically addresses

> use tdd-test-writer before implementing email validation
# → Writes comprehensive tests → Then implementation

> use code-quality-reviewer to review this PR
# → Objective analysis → No-praise feedback → Actionable items
```

## Agent Chaining

Agents delegate to each other automatically. Common flows:

**Ticket to PR:**
```
jira-workitem-implementer → feature-architect → tdd-test-writer → PR
```

**Quality pipeline:**
```
debugger → iterative-quality-fixer → code-quality-reviewer → gates pass
```

**PR review cycle:**
```
pr-review-comment-resolver → iterative-quality-fixer → pr-comment-validator → clean PR
```

## Auto-Activating Agents

These run proactively without being asked:

| Agent | Triggers when |
|---|---|
| **debugger** | Errors or test failures detected |
| **code-quality-reviewer** | After significant code changes |
| **iterative-quality-fixer** | Quality improvements needed |
| **jira-workitem-analyzer** | Working with Jira tickets |
| **information-consolidator** | Scattered content detected |
| **code-comment-reviewer** | During code reviews or refactoring |
| **tdd-test-writer** | Start of development tasks |
| **test-validator** | Test files need quality validation |

## Agent Types

| Type | Meaning |
|---|---|
| **Orchestrator** | Manages multi-step workflows, delegates to other agents |
| **Auto-activating** | Runs proactively when conditions detected |
| **Standard** | Called explicitly for specific tasks |

## Tool Access

| Level | Agents |
|---|---|
| Full (`*`) | feature-architect, jira-workitem-implementer |
| Broad | debugger, iterative-quality-fixer, tdd-test-writer, pr-review-comment-resolver |
| Read + analysis | code-quality-reviewer, test-validator, code-comment-reviewer |
| Specialized | datadog-log-searcher (Bash, WebFetch), meta-agent (Write, WebFetch, Firecrawl) |
